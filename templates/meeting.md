# 会议纪要拆解

适用：会议录音转写、飞书妙记导出、语音纪要——目标是让没参会的人 3 分钟接住全部信息，参会的人不漏任何待办。

规则：
- 决策和待办必须逐条穷尽，宁多勿漏；观点讨论可以概括
- 观点归属到人；转写没有说话人分离时按内容推断，开头声明「说话人归属由内容推断，可能有误差」
- 只写材料里真实存在的内容；转写有同音字误差时按上下文默默修正，人名/数字拿不准标「存疑」
- 没有明确结论的议题如实写「未达成结论」，不要替会议编一个决定

输出结构：

```
# 会议主题（自拟，一句话概括）
> 日期（如可知）| 参与者（如可知）| 时长

**一句话**：这场会议决定了什么 / 卡在了哪

## 决策（穷尽列出）
- 【决定】…（谁拍的板，如可知）
- 没有决策就写"本次会议无正式决策"

## 待办（穷尽列出）
- [ ] 事项 — 负责人（如提到）— 时间点（如提到）

## 议题与讨论
按实际议题分小节：各方观点（归属到人）、分歧点、未达成结论的地方

## 值得记住的原话
最多 3 条直引（带时间戳如有）

## 遗留问题
下次要继续聊的、材料里悬而未决的
```
